# 洞察信息日报系统 —— 需求文档

> 版本：v1.0  
> 日期：2026年4月7日  
> 状态：开发中

---

## 一、项目概述

### 1.1 项目目标

构建一套自动化新能源汽车行业信息采集与分析系统，每日从多个公开数据源抓取新闻，经关键词筛选、层面分类、LLM智能分析后，生成结构化的《洞察信息收集日报》，为岚图汽车战略部门提供决策参考。

### 1.2 核心用户

岚图汽车战略岗人员。

### 1.3 关键约束

- 日报中**不收录岚图自身的新闻**，聚焦外部环境信息
- 每日日报**最多20条**新闻，超出时按与新能源汽车/整车的相关性排序筛选
- 新闻的**发布日期**必须与日报日期匹配，无法确定日期的文章默认丢弃
- 开发阶段除明确要求"完整生成"外，默认使用 `--dry-run` 模式测试

## 二、日报格式与结构

### 2.1 输出格式

- 同时生成 **Word (.docx)** 和 **Markdown (.md)** 两种格式
- 文件命名：`洞察信息收集日报_YYYYMMDD.docx / .md`

### 2.2 文档结构（按顺序）

| 序号 | 模块 | 说明 |
|:---:|------|------|
| 1 | **标题 + 日期** | 居中显示"洞察信息收集日报"，下方居中显示日期（YYYY年MM月DD日，灰色字体） |
| 2 | **今日要点概括** | 紧跟标题之后，编号列表，每条50字左右，由LLM生成 |
| 3 | **四层面分类新闻表格** | 按四个层面分组，每组一个子标题 + 表格 |
| 4 | **初步分析与预警** | 另起一页，含战略意义、风险预警、近期关注 |

### 2.3 四层面分类

| 层面 | 子标题 | 覆盖内容 |
|------|--------|---------|
| 国家/政策层 | 国家政策导向 | 宏观政策、法规、补贴、关税、行业标准 |
| 行业/市场层 | 行业市场动态 | 市场份额、渗透率、价格战、行业报告、出海趋势 |
| 技术/研发层 | 电池、智能驾驶、智能座舱、芯片、新材料等领域的技术突破 | 储能、光伏、电池技术、智能驾驶、充电技术 |
| 业务/竞争层 | 车企动作 | 新车发布/上市/交付、企业战略、人事变动、合作并购 |

### 2.4 新闻表格字段（7列）

| 列名 | 列宽 | 字数要求 |
|------|------|---------|
| 时间 | 1.2cm | MM.DD格式，优先使用文章发布日期 |
| 事件内容 | 4.0cm | 50-70字，简洁精炼，含关键数据 |
| 参与方 | 1.8cm | 涉及的企业/机构/部门 |
| 事件影响 | 3.5cm | 50-70字，有分析深度 |
| 事件洞察 | 3.5cm | 50-70字，有分析深度 |
| 对岚图的影响及启示 | 3.0cm | 不超过40字，具体可执行 |
| 信息来源 | 2.5cm | 来源网站名称（Markdown中为超链接） |

### 2.5 初步分析与预警

| 子板块 | 字数要求 |
|--------|---------|
| 战略意义 | 100-150字 |
| 风险预警 | 100-150字 |
| 近期关注 | 每条50字左右 |

### 2.6 样式要求

- Word文档默认字体：**微软雅黑**，正文10号字
- 标题居中，日期居中灰色字体

## 三、新闻筛选规则

### 3.1 两级关键词匹配

| 级别 | 规则 | 示例 |
|------|------|------|
| **强相关词** | 命中**1个**即通过 | "新能源汽车"、"比亚迪"、"固态电池"、"智能驾驶"、"充电桩" |
| **弱相关词** | 需命中**2个**才通过 | "销量"、"芯片"、"AI"、"MPV"、"发布会" |

### 3.2 排除规则

命中任一排除词即**直接丢弃**：

| 类别 | 排除词示例 |
|------|-----------|
| **岚图自身** | 岚图、东风岚图、VOYAH、voyah |
| 燃油车领域 | 燃油车、发动机、变速箱、商用车、重卡、摩托车 |
| 非汽车领域 | 二手车、汽车金融、驾考、润滑油、汽车美容 |
| 消费电子 | 手机、平板、笔记本、耳机、智能手表 |
| 娱乐内容 | 游戏、影视、综艺、音乐、直播、短视频 |
| 金融股市 | 美股、盘前、盘后、涨跌、股价 |
| 其他无关 | 港口物流、洗地机、家电、假期旅游 |

### 3.3 标题过滤

- 标题少于**8个字符**的文章直接过滤（视为无意义标题）
- 自动清理标题前缀（如"中国储能网 -今日头条 - "）

### 3.4 相关性评分

用于排序和截取，按关键词类型和位置加权：

| 条件 | 得分 |
|------|------|
| 岚图/东风相关词在标题中 | +10 |
| 竞品词在标题中 | +7 |
| 通用词在标题中 | +5 |
| 岚图/东风相关词在正文中 | +5 |
| 竞品词在正文中 | +3 |
| 通用词在正文中 | +1 |

### 3.5 分类优先级

**政策层 > 技术层 > 公司层 > 得分判断 > 默认行业/市场层**

1. 命中国家/政策层关键词 → 国家/政策层
2. 命中技术/研发层关键词 → 技术/研发层
3. 命中具体公司/品牌关键词 → 业务/竞争层
4. 按各层面关键词得分比较，取最高分
5. 以上均未命中 → 默认归入"行业/市场层"

## 四、数据源配置

### 4.1 数据源分类

| 渠道类别 | 数据源 | 类型 | 状态 |
|---------|--------|------|------|
| **RSS源** | 36氪、IT之家 | RSS | ✅ 已启用 |
| **垂直媒体** | 盖世汽车、第一电动网、OFweek新能源汽车 | 网页爬取 | ✅ 已启用 |
| **综合门户** | 网易汽车、搜狐汽车 | 网页爬取 | ✅ 已启用 |
| **政府/协会** | 工信部 | 网页爬取 | ⚠️ 待优化 |
| **能源/储能/光伏** | 中国储能网、北极星储能网、国际能源网、每日光伏新闻、央视网-汽车、北极星电力网 | 网页爬取 | ✅ 已启用 |
| **手动补充** | 用户通过 `--urls` 参数输入 | 手动 | 按需 |

### 4.2 数据源策略

> **垂直媒体优先 + 综合门户覆盖 + 财经RSS补充 + 能源媒体补充政策层和技术层**

### 4.3 技术要求

- 每个数据源支持自定义CSS选择器（`list_selector`、`title_selector`、`content_selector`）
- 支持CSS属性选择器语法（如 `a[href*='/news/show-']`）
- 全局URL去重
- 请求间隔默认2秒
- 每个源最多抓取20条

## 五、日期过滤规则

### 5.1 核心原则

**新闻的发布日期必须与日报日期匹配**，以发布日期（通讯日期）作为加入日报的依据。

### 5.2 时间范围

- 默认抓取前**24小时**内的新闻
- 生成历史日报时可扩展至**72-96小时**

### 5.3 日期提取优先级

1. HTML时间元素（`.pub-time`、`.publish-time`、`time`等）
2. Meta标签（`article:published_time`等）
3. **URL路径中提取**（支持多种格式）：
   - `/YYYY/MM/DD/`（如搜狐 `/2026/04/03/`）
   - `/YYYYMM/DD/`（如每日光伏 `/202604/03/`）
   - `/YYYYMMDD/`（如北极星 `/20260403/`）
   - `-YYYYMMDD-`
4. 仅含年月的URL（如盖世汽车 `/202604/`）→ 设为当月1日

### 5.4 无日期处理

- **无法确定发布日期的文章默认丢弃**，避免旧新闻混入

## 六、文章数量限制

| 限制项 | 数值 | 说明 |
|--------|------|------|
| 每个数据源抓取上限 | **20条** | `max_articles_per_source` |
| 日报总条数上限 | **20条** | 超出时按相关性评分降序截取 |
| 正文截取长度 | **500字** | 用于筛选和分析 |
| 备用正文长度 | **3000字** | 正文选择器无效时的兜底 |

## 七、LLM分析需求

### 7.1 分析内容

每条新闻经LLM分析，提取以下结构化字段：
- 事件内容（50-70字）
- 参与方
- 事件影响/洞察（50-70字）
- 对岚图的影响及启示（不超过40字）

### 7.2 日报总结

基于所有分析结果生成：
- 要点概括（每条50字左右）
- 战略意义（100-150字）
- 风险预警（100-150字）
- 近期关注（每条50字左右）

### 7.3 技术配置

| 配置项 | 值 |
|--------|-----|
| LLM模型 | DeepSeek Chat (`deepseek-chat`) |
| 温度 | 0.3（偏向确定性输出） |
| 最大Token | 2000 |
| 请求间隔 | 1秒 |
| 降级模式 | LLM未配置时使用模板填充 |

## 八、系统功能

### 8.1 命令行接口

| 命令 | 说明 |
|------|------|
| `daily` | 生成日报（支持 `--date`、`--urls`、`--dry-run` 参数） |
| `biweekly` | 生成半月报 |
| `mark` | 标记重要新闻 |
| `status` | 查看配置状态 |
| `schedule` | 定时调度 |

### 8.2 关键参数

| 参数 | 说明 |
|------|------|
| `--date YYYYMMDD` | 指定日报日期 |
| `--urls URL1 URL2 ...` | 手动添加新闻URL |
| `--dry-run` | 仅展示筛选结果，不生成日报 |

### 8.3 质量保障

- **四层面覆盖检查**：生成后检查四个层面是否都有新闻，缺失时发出警告
- **人工审核标记**：未通过筛选但标题较长的文章标记为 `needs_review`

### 8.4 通知推送

- 支持企业微信Webhook
- 支持邮件发送

### 8.5 日志

- 所有操作记录到 `logs/app.log`
- 支持日志轮转（10MB，保留5个备份）

## 九、技术架构概览

```
数据源（RSS/网页/手动）
        ↓
    抓取模块（RSSFetcher / WebFetcher / ManualFetcher）
        ↓
    去重 + 日期过滤
        ↓
    关键词筛选（强相关/弱相关两级匹配 + 排除词）
        ↓
    层面分类（政策 > 技术 > 公司 > 得分）
        ↓
    相关性排序 + 20条截取
        ↓
    LLM分析（DeepSeek API）
        ↓
    日报生成（Word + Markdown）
        ↓
    通知推送（企业微信/邮件）
```
